Operazioni della ALU in ordine di codifica (A = 000, A OR B = 111)
[A; B; !B; B+1; A+B; A-B; A AND B; A OR B]

4bit di codice operativo + 8bit di dati

1011 MOV    register(Δ),register(B)        descr: dest ← source

Za: 0 JZ: 0 ALU:001 SH: 00 A:000 B: C:000000 M:000
0000 0100 0000 0000 0000 ⇒ 04000
mappa sorgente (B): 000=MDR; 001=PC; 010=GENERAL; 011=H; 100=D
mappa destinazione (Δ): 000=MAR; 001=MDR; 010=PC; 011=GENERAL; 100=H

memory

1100 STOR            MAR ↑ MDR

Za: 0 JZ: 0 ALU: 000 SH: 00 A: 000 B: C: 000000 M: 010
0000 0000 0000 0000 0010 ⇒ 00002

Deve essere anticipata da una MOV 00000BBB (MOV MAR SOURCE) ha lo scopo di abilitare la scrittura in RAM e scrivere

1101 LOAD                MAR ↓ MDR

Za: 0 JZ: 0 ALU: 000 SH: 00 A: 000 B: C: 000000 M: 100
0000 0000 0000 0000 0100 ⇒ 0004

Deve essere anticipata da una MOV 00000BBB ha lo scopo di abilitare la lettura dalla RAM e scrivere in MDR
